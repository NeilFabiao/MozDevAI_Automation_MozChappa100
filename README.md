---

## 🎯 Pitch da Sessão

A sessão começa com o **Excel como ponto de entrada dos dados**, explicando que na vida real os dados chegam de várias fontes e formatos, muitas vezes desorganizados.

Depois fazemos a **ingestão de 3 ficheiros diferentes**, simulando um cenário real de operação da MozChapa100 dentro do ecossistema MozBikes.

---

## 🛠️ Ferramentas (Tools) Utilizadas

Ao longo do processo usamos diferentes ferramentas:

- **Excel** → ponto de entrada e dashboard final  
- **Python (Jupyter Notebook)** → manipulação, limpeza e automação   
- **Pandas & sqlite3 (SQL Engine)** → análise rápida, joins, agregações e validação  
- **IA (Gemini / Copilot / ChatGPT)** → apoio na resolução de problemas e geração de código  

---

# 🧠 NEIL — Construção do Pipeline (MozChapa100 Analytics)

Os estudantes vão acompanhar **NEIL**, responsável pela construção técnica do pipeline de dados.

---

## 1. Ingestão de Dados
- Carregar 3 ficheiros Excel  
- Simular dados reais de operações da MozChapa100  

---

## 2. Estruturação com SQL no Jupyter Notebook

NEIL dar 5 min para os estudentes criarem de um script para organizar e preparar os dados para análise.

Inclui:
- criação de estrutura de dados  
- tratamento de missing values  
- tratamento de segmentos e inconsistências  

---

## ⚡ Desafio (Vibe Coding — NEIL)

Os alunos acompanham ter 10min na criação do script de ingestão de dados e regras de qualidade:

- remover valores negativos  
- garantir consistência de IDs  
- reforçar validação de métricas  

👉 Aqui a IA é usada como **copiloto de programação (Vibe Coding)**.

---

# 📊 STELA — Dashboard Final em Excel (MozChapa100 Operations)

Os dados finais são exportados para Excel, onde **STELA** assume a componente de análise e visualização.

---

## 📈 Trabalho de STELA

- construção do dashboard analítico no Excel  
- criação de análises Month-on-Month  
- identificação de top performers  
- identificação de bottom performers  
- análise de tendências operacionais  

---

## 💡 Resultado Esperado (NEIL & STELA)

No final, NEIL e STELA conduzem a fase de insights.

Os estudantes devem interpretar os dados e apresentar:

- insights de negócio  
- problemas operacionais  
- recomendações estratégicas para MozChapa100 / MozBikes  

---

# 🚀 Resumo

👉 Um pipeline completo de dados:

**Excel → Python → SQL → IA → Excel Dashboard**

Transformando dados brutos da MozChapa100 em **decisões inteligentes para o grupo MozBikes através de IA e automação**.
